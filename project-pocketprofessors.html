PocketProfessors

Trading-card inspired web app for study cards, packs, and collections

Overview

PocketProfessors is a MERN-style project inspired by the feel of the Pokémon TCG. Users open packs, collect “professor” cards, and track progress toward completing a set. I led front-end UX and collaborated on API design to make the experience fast, responsive, and accessible.

Key Features

  • Weighted pack opening: Cards drawn from the Cards collection with rarity weights (common → legendary), emulating TCG odds.
  • Collection tracking: User pulls stored in UserCards by UserId and card name; separate endpoints to fetch found vs unfound cards.
  • JWT refresh on each request: API returns a refreshed JWT token so the client always stays authenticated without extra round-trips.
  • Clean UX: Card grid with responsive layout, mobile-first touch targets, keyboard focus states, and aria-labels for screen readers.

Architecture

  • Backend: Node.js + Express REST API
  • Database: MongoDB (Cards and UserCards collections)
  • Core endpoints:
    • POST /api/openPack — apply rarity weights, return drawn cards, update UserCards
    • GET /api/foundCards — user’s collected cards
    • GET /api/unfoundCards — remaining cards to chase
    Each response refreshes and returns the user’s JWT.
  • Front-end: HTML/CSS/JS (semantic structure, responsive layout, ARIA for accessibility)

My Role & Contributions

  • Designed the landing & collection UIs (grid system, card components, hover/focus affordances).
  • Implemented pack-opening flow and client-side state updates after API responses.
  • Co-designed API shapes and error handling (e.g., token refresh, idempotent pulls).

Challenges & Lessons

  • Weighted randomness: Tuned rarity weights so pulls “feel right” while remaining statistically fair.
  • Token lifecycle: Simplified auth by letting each API call return a fresh JWT, reducing expired-token edge cases.
  • Performance + accessibility: Lazy-loaded media and ensured complete keyboard navigation with visible focus states.

Skills Learned

  • REST design and JSON contracts
  • Stateful UI patterns without a framework
  • Accessible components (ARIA roles, focus management)
  • MongoDB data modeling for collections/progress
