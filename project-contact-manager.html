Contact Manager

Semantic HTML/CSS/JS front end with a lightweight PHP backend

Overview

A clean, semantic, and responsive contact manager. The UI supports searching, adding, editing, and deleting contacts. Front end uses vanilla JavaScript (XMLHttpRequest) to interact with a PHP backend and updates the page without reloads.

Features

  • Live search: Debounced search box calls search.php and renders results dynamically.
  • CRUD: add.php, update.php, delete.php endpoints; optimistic UI updates with error fallback.
  • Form validation: HTML constraints (e.g., required, type="email", pattern) plus JS checks and accessible error messaging.
  • Responsive layout: Works on mobile and desktop with flexible forms and table/list views.

Architecture

  • Front-end: Semantic HTML sections, responsive CSS (flex/grid), unobtrusive JS for DOM updates.
  • Transport: XMLHttpRequest to PHP endpoints, JSON payloads, try/catch error handling.
  • Backend: PHP scripts (search.php, add.php, update.php, delete.php) responding with JSON.

My Role & Contributions

  • Implemented the delete flow and fixed a null-DOM reference bug (ensured elements are selected after render and guarded with null checks).
  • Wired up XHR helpers for GET/POST, standardized success/error handling, and status toasts.
  • Built accessible forms (labels/ids match, descriptive button text, keyboard focus order, visible focus rings).

Challenges & Lessons

  • Null element references: Resolved race conditions by deferring DOM queries until after content mount and guarding with existence checks.
  • Validation UX: Combined native constraints with custom messages so screen readers announce issues clearly.
  • Separation of concerns: Kept HTML semantic, CSS for layout/visuals, and JS strictly for behavior.

Skills Learned

  • Semantic HTML for forms and results lists
  • Accessible validation patterns (labels, aria-live regions)
  • AJAX with XMLHttpRequest and PHP endpoints
  • Responsive form design with flex/grid
